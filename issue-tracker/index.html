Issue Tracker

Use the public issue tracker for bugs and feature requests.

The support repository is the public issue tracker for Commit Guard. Use the links below so reports come in with the right template and context.

Bug

Report a bug

Use this when the plugin is broken, a workflow regressed, or actual behavior diverges from documented behavior.

  • Include IntelliJ version and plugin version
  • Include provider, edition, and operating system
  • Include reproduction steps and screenshots or logs if available
Open bug report
Feature

Request a feature

Use this when you want a new workflow capability, integration, settings option, or UX improvement.

  • Describe the workflow you want to improve
  • Explain what is missing today
  • Note alternatives and tradeoffs if you already considered them
Open feature request
Browse

Review open issues

Before filing a new issue, check if the problem or request already exists and add any extra context there.

  • Search by provider, workflow, or IntelliJ version
  • Confirm whether the issue is already tracked
  • Link related docs or screenshots when commenting
