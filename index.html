Public Support

Use the issue tracker for product feedback and reproducible problems.

The support repo is public so users, JetBrains reviewers, and your team all have one clear place to discuss bugs and requests.

Report a bug

Use the bug report template when something is broken, regressed, or behaving inconsistently.

  • Include IntelliJ version and plugin version
  • Include provider, edition, and reproduction steps
  • Attach logs or screenshots when possible
Open bug template

Request a feature

Use the feature request template when you want a new capability, workflow, or UX improvement.

  • Describe the workflow you are trying to enable
  • Explain what blocks you today
  • Note any alternatives you already tried
Open feature template

Before filing publicly

Check the docs first, then use the public issue tracker for product bugs, regressions, and enhancement requests.

  • Check the docs first for setup and expected behavior
  • Use the public issue tracker for product bugs and enhancements
  • Include reproduction steps, plugin version, and screenshots when possible
